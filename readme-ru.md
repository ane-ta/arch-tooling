# Arch Tooling 🛠️

Набор переиспользуемых go-task скриптов для подхода documentation-as-code.

Подключается к проекту, как **Git Submodule**.

---

## 🚀 Что внутри?
Запуск:
* **Structurizr** - локальный запуск Structurizr движка для визуализации модели.

CI/CD:
* **Git** - обновление сабмодулей по имени в директории репозиториев.

Экспорт:
* **Structurizr -> C4 Puml -> SVG** — экспорт архитектурных схем из `structurizr.dsl` в форматы PUML и SVG с использованием Docker-Pandoc. 
* **ORG -> MD** — экспорт директории с .org файлами в директорию с .md файлами с сохранением дерева поддиректорий с использованием Docker-Pandoc.

---

## 🛠️ Быстрый старт
Чтобы использовать эти инструменты в своем проекте

### 1. Подключение сабмодуля
Добавьте этот репозиторий в директорию `arch-tooling`:

```bash
git submodule add <URL_ЭТОГО_РЕПОЗИТОРИЯ> tools/arch-tooling
git submodule update --init --recursive
```

### 2. Кастомизация

В корневом Taskfile.yml вашего проекта подключите главный файл инструментов:
```yaml
version: '3'

includes:
  arch:
    taskfile: ./arch-tooling/scripts/Taskfile.yml
```
После этого вам станут доступны все открытые публичные задачи. Посмотреть их список и описание можно прямо в терминале:
```bash
task --list
```
